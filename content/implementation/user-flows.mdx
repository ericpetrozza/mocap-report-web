## Flow 1: Sample Request Flow

### Goal
Request free product samples to test fit before placing full order.

### Who This Flow Serves
First-time buyers (Problem Solver persona) who need to verify product compatibility before purchasing.

### High-Level Flow Diagram

```
◼ Google Search Results
→ User lands on application landing page
◼ Powder Coating Protection Landing Page
→ User clicks "Request Free Samples"
◼ Sample Request Page
◆ Does user know exact size?
 → Yes: Select specific SKU from table
 → No: Use Size Guide Helper
◼ Size Guide Helper (if needed)
→ User adds 3-5 samples to cart ($0.00 each)
◼ Cart Page
◆ Sample only or add full order?
 → Sample only: proceed to checkout
 → Add products: continue shopping
◼ Checkout (guest OK, shipping $12.95)
→ Complete order
◼ Order Confirmation
★ Success: Samples will arrive in 5-7 days
```

### Critical Breakpoints
- **Size uncertainty**: User doesn't know which size fits their application
- **Guest vs account**: User must decide whether to create account
- **Shipping cost surprise**: Sample shipping fee must be transparent upfront
- **Follow-up conversion**: Sample arrival doesn't automatically convert to sale

### What the System Must Support
- $0.00 cart items (free sample products)
- Size guide helper tool with measurement instructions
- Guest checkout (no forced registration)
- Shipping cost calculator showing $12.95 upfront
- Email reminder system (2 weeks post-delivery)
- Easy path from sample confirmation back to full order

### Friction Summary
- No clear sample request process exists
- Users must call or email sales rep
- Lead time and availability unclear

### Opportunity Summary
- Self-service sample ordering reduces sales calls
- Size helper reduces wrong selections
- Email follow-up automates conversion nurturing
- Guest checkout removes registration friction
- Clear shipping cost prevents cart abandonment

## Flow 2: Product Browsing Flow

### Goal
Find the correct protective cap for hydraulic fittings by browsing application-specific recommendations.

### Who This Flow Serves
Engineers (Explorer persona) who prefer to browse by application rather than search by part number.

### High-Level Flow Diagram

```
◼ Homepage
◆ How does user want to browse?
 → By Application (selected)
 → By Product Type
 → By Industry
 → By Part Number
◼ Applications Page
→ User selects "Hydraulic Fitting Protection"
◼ Application Landing Page
→ User selects "Threaded Vinyl Caps"
◼ Product Family Page (Interactive Table)
→ User applies filters: NPT + 3/4"
◼ Filtered Results (12 products)
→ User selects 2 items to compare
◼ Comparison View
◆ Which variant is right?
 → Red selected (visibility for QC)
◼ Quantity Modal
→ User enters quantity: 1,000 pcs
◼ Add to Cart Confirmation
★ Success: Product identified and added to cart
```

### Critical Breakpoints
- **Navigation choice**: User must find application-based entry point
- **Filter overwhelm**: 320+ variants need smart filtering
- **Size verification**: User needs confidence they selected correct size
- **Quantity decision**: Standard packaging vs custom quantity
- **Comparison paralysis**: Too many similar options

### What the System Must Support
- Multi-path navigation (application, product type, industry, part number)
- Interactive filterable product tables
- Live filter counts showing results before applying
- Side-by-side comparison tool (2-5 products)
- Inline "Add to Cart" actions in tables
- Quantity modal with pricing updates
- Toast notifications for cart actions

### Friction Summary
- No application-based entry path exists
- Static HTML tables cannot be filtered or sorted
- No comparison functionality between variants

### Opportunity Summary
- Application navigation matches user mental models
- Interactive tables reduce cognitive load
- Comparison tool reduces uncertainty
- Inline cart actions improve efficiency
- Real-time price updates increase transparency

## Flow 4: Size Chart Interpretation Flow

### Goal
Identify correct product size by measuring application and interpreting size chart.

### Who This Flow Serves
Procurement managers (Veteran persona) who need to match product dimensions to their specific application.

### High-Level Flow Diagram

```
◼ Product Family Page
→ User sees "Not sure which size? Measure your application"
→ User clicks "How to Measure"
◼ Measurement Guide Modal
→ Visual diagram shows ID, OD, Length
→ Step-by-step measurement instructions
→ User measures application: 3/4" NPT thread
◼ Interactive Size Table
→ User filters: Size = 3/4", Thread Type = NPT
◼ Filtered Results (4 variants)
◆ Does size look correct?
 → Uncertain: use Size Confirmation Helper
 → Confident: add to cart
◼ Size Confirmation Helper (if needed)
→ System verifies fit based on measurements
→ Recommendation provided with explanation
◆ Still uncertain?
 → Yes: Request free sample
 → No: Add to cart
★ Success: Correct size selected with confidence
```

### Critical Breakpoints
- **Measurement knowledge**: User may not know how to measure properly
- **Unit confusion**: Metric vs imperial measurements
- **Fit tolerance**: Understanding how tight/loose fit should be
- **Verification anxiety**: Uncertainty despite measurement
- **Wrong selection recovery**: Easy returns if size incorrect

### What the System Must Support
- Visual measurement guide with diagrams (ID, OD, Length)
- Metric/imperial converter tool
- Interactive size helper asking clarifying questions
- Fit verification with specific explanations
- Sample request option from size selector
- Easy returns process for wrong sizes

### Friction Summary
- Size charts are static HTML with no guidance
- No measurement instructions or visual aids
- Customers frequently guess wrong and return

### Opportunity Summary
- Visual guides reduce measurement errors
- Interactive helper builds confidence
- Sample option provides verification path
- Clear explanations prevent returns
- Easy returns policy reduces purchase anxiety

## Flow 6: Search & Filter Refinement Flow

### Goal
Find exact product variant quickly using search with intelligent autocomplete and filtering.

### Who This Flow Serves
Power users (Veteran persona) who know what they want and expect fast, precise search.

### High-Level Flow Diagram

```
◼ Homepage
→ User types in search: "3/4 vinyl cap red"
◼ Search Autocomplete Appears
→ Shows product suggestions + past queries
→ User selects autocomplete suggestion
◼ Search Results Page
→ Smart filters auto-applied from query
→ Filters shown: Size=3/4", Material=Vinyl, Color=Red
→ 147 results displayed
⚠ Still too many results
→ User adds filter: Pull Tab feature
◼ Refined Results (34 products)
→ User changes sort: Price Low to High
→ Top result: RVC0750RPT
→ User clicks inline "Add to Cart"
◼ Quantity Modal
→ User enters 5,000 pcs
→ Volume discount auto-applies
★ Success: Exact variant found in under 30 seconds
```

### Critical Breakpoints
- **Search quality**: Query must return relevant results
- **Filter intelligence**: System should auto-apply obvious filters
- **Refinement clarity**: Live counts show impact of filters before applying
- **Sort priority**: Default sort may not match user preference
- **Cart friction**: Multi-step process slows power users

### What the System Must Support
- Smart autocomplete (queries + products + categories)
- Natural language query parsing
- Auto-applied filters from search terms
- Live filter counts showing result numbers
- Multiple sort options (price, popularity, stock)
- Inline "Add to Cart" actions
- Search analytics capturing: query, filters, sort, time-to-cart, conversion

### Friction Summary
- Search returns zero results for most queries
- No autocomplete suggestions
- No filter refinement capability
- Static results with no sorting

### Opportunity Summary
- Smart autocomplete reduces typing and errors
- Auto-applied filters save clicks
- Live counts prevent dead-end filtering
- Inline cart actions improve efficiency
- Analytics enable continuous search improvement
